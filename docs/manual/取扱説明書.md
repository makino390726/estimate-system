## 1. はじめに

### 1.1 システムの目的

見積書・案件・注文・商品マスタに加え、**修理受付から完了報告・事務売上確認**までを一元的に扱う Web アプリケーションです。

### 1.2 推奨ブラウザ

- Google Chrome / Microsoft Edge（最新版）
- 修理現場のスマホ画面は `/repair-mobile` を利用

### 1.3 本番 URL

- 本番: `https://estimate-system-ten.vercel.app`
- ローカル開発: `http://localhost:3000`

## 7. 修理案件管理（概要）

### 7.1 修理案件管理（PC）

修理受付〜ステータス管理〜完了報告の中心画面です。

![修理案件管理](screenshots/20-repair-requests.png)

**主なステータス（本システムで扱う範囲）**

| ステータス | 意味 |
|------------|------|
| 受付 | 新規依頼直後 |
| 担当者確認 | 営業担当が確認済み |
| 修理中 | 作業中 |
| 完了報告済 | 現場から完了報告送信済み |

一覧から案件を開き、詳細・部品・完了報告を操作します。

### 7.2 部品等売上処理（事務）

完了報告済み案件の**出張料・工賃・部品**を確認し、事務が**確認ボタン**を押して売上処理に進みます。

![部品等売上処理](screenshots/21-repair-sales-processing.png)

| 表示 | 意味 |
|------|------|
| 赤「確認」 | 未確認 → クリックで確認済みにする |
| 緑「確認済」 | 事務確認完了 |

**未確認のみ** チェックで絞り込み可能です。LINE WORKS 通知のリンク（`?focus=案件ID`）から開くと、該当行がハイライトされます。

### 7.3 機械カルテ・故障分析

![機械カルテ](screenshots/22-machine-cards.png)

![故障分析](screenshots/23-repair-dashboard.png)

### 7.4 修理対応（現場スマホ）

現場担当者向け。案件一覧から個別案件を開き、症状・部品・出張料・**完了報告**を入力します。

![修理対応一覧](screenshots/24-repair-mobile.png)

**完了報告の流れ（現場）**

1. 案件を開く
2. 出張料・工賃・部品を入力・保存
3. **完了報告を送信** → ステータスが完了報告済みに
4. 顧客へ LINE 通知（LINE 受付案件の場合）
5. 営業担当・**事務担当**へ LINE WORKS 通知

## 8. 修理フロー詳細（LINE / LINE WORKS）

### 8.1 全体フロー

```mermaid
flowchart LR
    A[顧客 LINE 依頼] --> B[受付]
    B --> C[営業 LINE WORKS 通知]
    C --> D[担当者確認 / 修理中]
    D --> E[現場 完了報告]
    E --> F[顧客 LINE 完了通知]
    E --> G[事務 LINE WORKS 通知]
    G --> H[部品等売上処理で確認]
```

### 8.2 顧客向け LINE 修理依頼（LIFF）

LINE 上の Web フォームから修理を依頼します。

![LINE修理依頼フォーム](screenshots/28-liff-repair-form.png)

※ 実際は LINE 公式アカウントの LIFF からスマホで開きます（説明書の画面はイメージ図です）。

### 8.3 担当者通知チャネル

| モード | 画面 | 用途 |
|--------|------|------|
| LINE WORKS（推奨） | 修理通知 LINE WORKS 連携 | 営業所担当・事務担当 |
| LINE 公式 | 修理通知 LINE 連携 | 代理店等（別運用） |

**LINE WORKS と LINE 公式は併用しません。** 環境変数 `REPAIR_STAFF_NOTIFY_CHANNEL` で切替えます。

### 8.4 LINE WORKS 連携の登録

![LINE WORKS連携](screenshots/25-lineworks-staff-notify.png)

**登録手順（推奨）**

1. **登録する担当者** をプルダウンから選択（手入力しない）
2. QR を担当者のスマホで読み取り、ログインメールを登録  
   または **詳細 → 手動登録** で LINE WORKS ID（メール）を保存
3. 連携状態で **APIトークン OK**・**Supabase service role 設定済み** を確認

![LINE WORKS自己登録](screenshots/27-lineworks-staff-register.png)

**よくあるエラー**

- `担当者が staffs に見つかりません` → プルダウンから担当者を選び直す
- `office_sales_confirmed_at does not exist` → Supabase で `add_repair_office_sales_confirmed.sql` を実行

### 8.5 LINE 公式連携（参考）

LINE 公式アカウント向けの担当者連携です（LINE WORKS モード時は通常未使用）。

![LINE公式連携](screenshots/26-line-staff-notify.png)

### 8.6 事務向け完了通知の文面

完了報告時、事務担当には次の趣旨が LINE WORKS で届きます。

> 担当者より修理完了報告がなされました。  
> システムを確認のうえ、確認ボタンをクリックしてから売上処理を行ってください。

ボタン **確認・売上処理** から部品等売上処理画面へ遷移します。

## 9. よくある質問

### Q1. メニューに戻るには？

各画面の **← メニュー** / **メインメニュー** リンク、または URL で `/selectors` を開きます。

### Q2. 修理の新規バッジは？

メインメニューの「修理案件管理」に、ステータス「受付」の件数が表示されます。

### Q3. 画面キャプチャを更新したい

リポジトリで `npm run docs:capture` を実行（`docs/manual/README.md` 参照）。

### Q4. 管理者向け DB 設定

| SQL ファイル | 内容 |
|--------------|------|
| `add_repair_office_staff_notify.sql` | 事務担当・担当事業所 |
| `add_repair_office_sales_confirmed.sql` | 事務確認日時 |
| `apply_repair_prerequisites.sql` | 修理関連の一括前提 |

Supabase SQL Editor で実行してください。
